# Prompts de seguimiento para Codex

## 1. Mejorar conversión
Revisa la landing como especialista en conversión para servicios locales en Costa Rica. El objetivo principal es que el visitante escriba por WhatsApp. Mejora claridad, CTAs, orden de secciones y textos sin cambiar el stack ni agregar backend.

## 2. Mejorar diseño visual según marca
Refina la interfaz para que se parezca más al board visual de FENRIR: limpia, formal, geométrica, navy/azul, tarjetas blancas, fondos claros, bloques oscuros elegantes y mucho espacio. No uses estilo gamer, fantasía ni lobo.

## 3. QA técnico
Haz una revisión técnica completa del proyecto. Busca problemas de responsive design, accesibilidad, SEO, performance, links rotos, botones de WhatsApp, estructura semántica HTML, imágenes sin alt, código duplicado y componentes innecesarios. Devuélveme hallazgos y aplica correcciones.

## 4. Preparar despliegue
Prepara el proyecto para Cloudflare Pages y Netlify. Incluye build command, output directory, configuración necesaria, README paso a paso, favicons, Open Graph y validación de `npm run build`.

## 5. Crear demos más realistas
Mejora las rutas `/demos/soda/`, `/demos/barberia/` y `/demos/servicio-tecnico/` para que parezcan demos reales de negocios en Costa Rica. Mantén nombres ficticios, evita datos reales y usa contenido realista. Cada demo debe tener hero, servicios, horarios, ubicación ficticia, galería placeholder, WhatsApp y CTA final.

## 6. Preparar mensajes de WhatsApp
Centraliza los mensajes de WhatsApp por paquete y por demo. Cada botón debe abrir WhatsApp con un mensaje prellenado distinto, por ejemplo: “Hola, vi el paquete Página Básica y quiero información para mi negocio.”

## 7. Revisar copy para clientes no técnicos
Reescribe todos los textos para que sean claros para dueños de negocios no técnicos. Evita palabras como stack, frontend, hosting estático, deploy, framework, componente o tecnología. Habla en términos de negocio: contactos, confianza, servicios, ubicación, horarios y clientes.
